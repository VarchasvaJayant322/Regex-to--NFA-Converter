ε What is an ε-NFA?

A Nondeterministic Finite Automaton with ε-transitions is a state machine where the machine can simultaneously be in multiple states, and can transition between states without consuming any input (via ε-moves).

States: {q0, q1, q2}
Alphabet: {a, b}
ε-moves allow "free" jumps

⚙ Thompson's Construction

A recursive algorithm that builds an ε-NFA from a regular expression. Each operator (|, *, +, ?) has a corresponding NFA fragment that gets composed together. Every fragment has exactly one start and one accept state.

RE → Parse → NFA fragments → Compose

∪ Union: A|B

Creates a new start state with ε-transitions to both A and B's starts. Creates a new accept state reachable via ε from both A and B's accept states. The machine nondeterministically picks a branch.

q_new →ε→ q_A_start
q_new →ε→ q_B_start
q_A_end →ε→ q_accept

· Concatenation: AB

Connect A's accept state to B's start state with an ε-transition. The result uses A's start state and B's accept state. This sequences the two automata.

q_A_start → … → q_A_end
q_A_end →ε→ q_B_start
q_B_start → … → q_B_end

★ Kleene Star: A*

Add a new start and accept state. The new start connects via ε to A's start and directly to the new accept (allowing zero repetitions). A's accept loops back to A's start for repeated matching.

q_new →ε→ q_A (or skip)
q_A_end →ε→ q_A_start (loop)
q_A_end →ε→ q_accept

ε ε-Closure

The ε-closure of a state S is the set of all states reachable from S by following zero or more ε-transitions. When processing input, we always expand the current state set to its full ε-closure first.

ε-closure({q0}) = {q0, q1, q3}
(via free ε-jumps from q0)
